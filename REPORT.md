# Практична робота №1

## Тема: Найпростіші класи. Інкапсуляція та властивості в C#
## Що таке інкапсуляція

Інкапсуляція - це принцип ООП, за якого внутрішній стан об'єкта приховується від прямого доступу, а зміни виконуються через контрольовані методи та властивості. Це допомагає захищати об'єкт від некоректних значень і концентрувати бізнес-правила всередині класу.

Приклади з коду:

1. У `Student` поле `_fullName` приватне, а доступ іде через властивість `FullName`, яка перевіряє, що ПІБ не порожнє і має мінімум 5 символів.
2. Поле `_recordBookNumber` приховане, а властивість `RecordBookNumber` має `init`, тому номер залікової задається під час створення студента і не змінюється випадково.
3. Поле `_averageGrade` змінюється тільки через приватний setter властивості `AverageGrade` і метод `UpdateAverageGrade`, де є перевірка діапазону 0-10 та округлення.
4. У `StudentGroup` список `_students` приватний. Зовні доступна тільки read-only проекція `Students`, а додавання й видалення виконуються через `AddStudent` і `RemoveStudent`.
5. У `GradeJournal` оцінки додаються через `AddOrUpdateGrade`, тому назва предмета і бал проходять валідацію перед записом у `Dictionary<string, double>`.

## Авто-властивості та властивості з повною реалізацією

Авто-властивості зручні, коли не потрібна додаткова логіка. У роботі це `Status`, бо статус можна змінювати напряму, і `EnrollmentDate`, бо дата зарахування задається при створенні.

Властивості з повною реалізацією потрібні, коли треба контролювати значення. Наприклад, `FullName`, `DateOfBirth`, `RecordBookNumber`, `AverageGrade`, `PersonalEmail`, `Notes`, `Name`, `Specialty`, `Course` мають backing fields і перевірки.

## Для яких даних застосована валідація

Валідація застосована для таких даних:

- `FullName`: не порожнє, мінімум 5 символів.
- `DateOfBirth`: не майбутня дата, вік не більше 120 років.
- `RecordBookNumber`: рівно 8 цифр.
- `AverageGrade`: число від 0 до 10, округлення до 2 знаків.
- `PersonalEmail`: базова перевірка формату email регулярним виразом.
- `Course`: курс від 1 до 6.
- `StudentGroup.AddStudent`: перевірка унікальності номера залікової в межах групи.
- `GradeJournal.AddOrUpdateGrade`: назва предмета не порожня, оцінка від 0 до 10.

## Read-only та init-only властивості

Read-only властивості потрібні для даних, які мають обчислюватися або тільки читатися ззовні. Приклад: `Age` у `Student` не зберігається окремо, а рахується через `CalculateAge`. Також `GroupSize` і `AverageGroupGrade` у `StudentGroup` обчислюються на основі приватного списку студентів.

Init-only властивості потрібні для даних, які можна задати під час створення об'єкта, але не варто змінювати після цього. Приклад: `RecordBookNumber` і `EnrollmentDate` у `Student`.

## Як працює Age

`Age` - обчислювана властивість. Вона викликає `CalculateAge`, який бере поточну дату, віднімає рік народження і додатково перевіряє, чи день народження вже був у поточному році. Вік не зберігається в полі, бо він змінюється з часом, навіть якщо дані студента не редагувалися.

## Принцип єдиної відповідальності

Кожен клас має свою зону відповідальності:

- `Student` описує одного студента, його дані, валідацію і прикладну логіку на рівні студента.
- `GradeJournal` відповідає тільки за оцінки по предметах і перерахунок середнього бала.
- `StudentGroup` керує колекцією студентів, пошуком, статистикою, фільтрацією та JSON-збереженням групи.
- `Program` відповідає за консольне меню і взаємодію з користувачем.

## Труднощі під час проєктування

Основна складність - поєднати вимогу "приватні поля для всіх даних" із JSON-серіалізацією. Щоб не відкривати внутрішній список студентів для прямої зміни, у `StudentGroup` використано приватний список `_students`, read-only властивість `Students` і окремий приватний record `StudentGroupStorage` для збереження у файл.
Ще одна складність - редагування номера залікової. Він зроблений `init-only`, бо це ідентифікатор студента. Тому в меню можна редагувати інші дані, а для зміни номера логічніше видалити студента і додати нового з правильним номером.

Під час виконання роботи закріплено використання різних типів властивостей у C#, валідацію у setter, обчислювані властивості, `required`, `init`, приватні setter, роботу з `List<T>`, `Dictionary<TKey,TValue>`, enum, JSON через `System.Text.Json`, а також базову Git-структуру з feature-гілками.

## Робота з Git

Репозиторій ініціалізовано в папці `Pracktuchna №1_Куцевич`. Створено `.gitignore`, щоб не додавати в Git службові папки `bin/`, `obj/`, `.vs/` і файл даних `students.json`.

Структура гілок відповідає методичці:

- `main` - стабільна фінальна версія практичної роботи.
- `feature/student-class` - гілка для етапу створення класу `Student`, enum `StudentStatus`, валідації, властивостей і `GradeJournal`.
- `feature/group-management` - гілка для етапу створення `StudentGroup`, пошуку, фільтрації, статистики та роботи зі списком студентів.
- `feature/file-storage` - гілка для етапу консольного меню, JSON-збереження та завантаження даних.

Кожен етап оформлено окремими комітами з описовими повідомленнями. Після створення репозиторію на GitHub треба додати remote і запушити кожну гілку окремо:

```bash
git remote add origin https://github.com/USERNAME/REPOSITORY.git
git push -u origin main
git push -u origin feature/student-class
git push -u origin feature/group-management
git push -u origin feature/file-storage
```
